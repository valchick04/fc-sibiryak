# ФК «Сибиряк» — сайт футбольного клуба

Django-проект официального сайта футбольного клуба «Сибиряк» из Братска. На сайте есть страницы матчей, билетов, магазина, команды, новостей, контактов, профиля пользователя и корзины.

## Основные возможности

- Главная страница клуба с новостями и информацией о матчах.
- Раздел «Матчи» с календарём, таблицей, составами и статистикой.
- Раздел «Билеты» с выбором матча, сектора, ряда и места.
- Раздел «Магазин» с товарами клуба.
- Корзина для товаров и билетов.
- Оформление заказа с выбором способа оплаты: **СБП** или **Карта**.
- После покупки:
  - у товаров уменьшается остаток `stock`;
  - билеты получают статус `sold`;
  - у билетов сохраняется покупатель и дата покупки;
  - в базе данных создаётся заказ и позиции заказа.
- Регистрация, вход, выход и профиль пользователя.
- Админ-панель Django для управления данными сайта.

> Важно: оплата в проекте учебная/демонстрационная. СБП и карта выбираются как способ оплаты, но реальный банк или эквайринг не подключены. После оформления заказ сразу считается оплаченным.

## Установка и запуск

### 1. Перейти в папку проекта

```bash
cd fc-sibiryak
```

### 2. Создать виртуальное окружение

```bash
python3 -m venv venv
```

### 3. Активировать виртуальное окружение

Для macOS / Linux:

```bash
source venv/bin/activate
```

Для Windows:

```bash
venv\Scripts\activate
```

### 4. Установить зависимости

```bash
pip install -r requirements.txt
```

### 5. Применить миграции

```bash
python3 manage.py migrate
```

### 6. Запустить сервер

```bash
python3 manage.py runserver
```

После запуска сайт будет доступен по адресу:

```text
http://127.0.0.1:8000/
```

Если порт `8000` занят, можно запустить проект на другом порту:

```bash
python3 manage.py runserver 8001
```

И открыть:

```text
http://127.0.0.1:8001/
```

## Админ-панель

Админка Django доступна по адресу:

```text
http://127.0.0.1:8000/admin/
```

Если суперпользователь ещё не создан, создай его командой:

```bash
python3 manage.py createsuperuser
```

После этого можно войти в админку и управлять товарами, билетами, заказами, новостями, матчами и пользователями.

## Корзина и покупка

Корзина находится по адресу:

```text
http://127.0.0.1:8000/cart/
```

Покупка работает так:

1. Пользователь добавляет товар из магазина или билет со страницы билетов.
2. Данные корзины сохраняются в `localStorage` браузера.
3. На странице корзины пользователь выбирает способ оплаты: **СБП** или **Карта**.
4. При нажатии кнопки оплаты отправляется запрос на серверный адрес:

```text
/api/checkout/
```

5. Сервер проверяет наличие товаров и доступность билетов.
6. Создаётся заказ в базе данных.
7. Остаток товара уменьшается.
8. Купленные билеты получают статус `Продан`.

## Основные модели

### Product

Модель товара магазина. Важные поля:

- `name` — название товара;
- `description` — описание;
- `price` — цена;
- `stock` — количество товара на складе;
- `is_active` — показывать ли товар на сайте.

### Ticket

Модель билета на матч. Важные поля:

- `match` — матч;
- `sector` — сектор;
- `row` — ряд;
- `seat` — место;
- `price` — цена;
- `status` — статус билета: доступен, зарезервирован или продан;
- `owner` — покупатель;
- `purchased_at` — дата покупки.

### Order

Модель заказа. Важные поля:

- `user` — пользователь;
- `status` — статус заказа;
- `payment_method` — способ оплаты: СБП или карта;
- `paid_at` — дата оплаты.

### OrderItem

Позиция заказа. Может быть связана с товаром или билетом.

## Частые проблемы

### Порт уже занят

Ошибка:

```text
Error: That port is already in use.
```

Решение — запустить на другом порту:

```bash
python3 manage.py runserver 8001
```

Или найти процесс на порту `8000`:

```bash
lsof -i :8000
```

Затем остановить его по PID:

```bash
kill -9 PID
```

Например:

```bash
kill -9 63183
```

### CSS не обновляется

Если изменения стилей не видны, обнови страницу без кэша:

```text
Cmd + Shift + R
```

Также можно временно изменить подключение CSS в шаблоне:

```html
<link rel="stylesheet" href="{% static 'club/css/site.css' %}?v=2">
```
